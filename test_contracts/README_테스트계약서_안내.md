# 테스트용 임대차계약서 (계약똑똑 분석 검증용)

국토교통부 **주택임대차표준계약서** 양식을 기반으로 만든 가상 계약서 8건입니다.
모든 인적사항(이름·주민번호·연락처·주소)과 단지명은 **가공의 더미 데이터**입니다.

- 형식: PDF (텍스트 기반 — 업로드 후 OCR/파싱 테스트용)
- 구성: 정상 2건 + 위험 6건 / 전세·월세·반전세 혼합
- 전세는 `차임(월세) 해당 없음`, 월세·반전세는 실제 차임 금액 표기 → 계약 유형 자동 감지 검증 가능

## 활용 팁

- **정상(01·02·05)**: 위험 조항이 거의 없어 결과가 대부분 "정상/안전"으로 나와야 합니다(오탐 점검용).
- **위험(03·04·06·07·08)**: 위 표의 특약이 "위험/주의"로 탐지되고, 관련 법령(주택임대차보호법·민법)이 근거로 제시되는지 확인하세요.
- 전세가율 검증: 03·04는 보증금이 높아 시세 대비 전세가율이 위험 범위로 나오는지 함께 확인하면 좋습니다.

## 사진 업로드(이미지 OCR) 테스트용 PNG

`images/` 폴더에 대표 6건(01·03·06·09·13·18)을 150dpi PNG로 렌더해 뒀다. 앱의 "사진 촬영/파일 선택"으로 업로드하면 이미지 OCR(구글 비전) 경로를 점검할 수 있다. (이미지 OCR은 GOOGLE 비전 자격증명이 설정돼 있어야 동작한다. PDF 경로는 pdfplumber라 자격증명 불필요.)
(유형 감지는 실제 `_detect_contract_type`가 "만원" 단위를 처리하므로 도커 실행 시 16/16. 로컬 간이 측정에선 15번만 만원 미파싱으로 1건 누락.)

재측정: `docker compose run --rm --no-deps -v "$PWD:/work" -w /work backend python scripts/measure_clause_risk.py`
