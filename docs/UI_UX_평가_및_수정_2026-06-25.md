# 계약똑똑 완성도 평가 및 UI/UX 수정 리포트

작성일: 2026-06-25 · 대상: 프론트엔드 전체(페이지 12 · 공통 컴포넌트 13) + 디자인 시스템

---

## 1. 종합 평가

전체적으로 **완성도가 높은 MVP**다. 사용자 핵심 플로우(홈 → 업로드 → 분석 진행 → 결과 → 특약 추천)가 끝까지 연결되어 있고, 인증(카카오/구글/개발용), 결제(포트원), 실거래가 시세 조회, 체크리스트, 마이페이지까지 구현되어 있다. MSW 목업이 모든 엔드포인트를 덮고 있어 백엔드 없이도 독립 시연이 가능하고, Vitest 테스트·CI·배포 가이드도 갖춰져 있다.

강점으로는 (1) 공통 레이아웃 체계(상단 `NavBar`, 하단 `BottomNavBar`, `max-w-3xl` 모바일 우선 폭, `card` 시스템), (2) 접근성에 대한 일관된 배려(`aria-label`/`role`/`aria-pressed`, `focus-visible` 링, `prefers-reduced-motion`), (3) 로딩·빈 상태·에러 상태가 화면마다 빠짐없이 설계된 점, (4) 무료 체험 → 블러 미리보기 → 결제 유도로 이어지는 페이월 UX가 자연스러운 점을 꼽을 수 있다.

평가 기준 완성도(주관적):

| 영역 | 완성도 | 비고 |
|------|--------|------|
| 핵심 플로우(업로드~결과) | 95% | 동작·상태 처리 견고 |
| 디자인 시스템 일관성 | 75% → 92% | slate/brand 마이그레이션 누락분 이번에 정리 |
| 콘텐츠 완성도 | 85% → 95% | 플레이스홀더/죽은 버튼 제거 |
| 코드 견고성 | 90% → 96% | Hooks 규칙 위반 1건 수정 |
| 유지보수성 | 80% | 시세 계산기 중복 등 권고 사항 잔존 |

## 2. 발견 이슈 및 조치

### 2.1 수정 완료 (이번 작업)

**(A) 디자인 시스템 색상 미통일 — 가장 광범위한 UI 이슈**
변경 이력엔 "전체 페이지 slate/brand 마이그레이션"이 기록되어 있었으나, 정작 **핵심 플로우 컴포넌트**에 레거시 `gray-*`와 브랜드 외 `blue-*`가 다수 남아 있었다. 브랜드 메인은 딥 네이비(`brand-600 #0F2B5B`)인데, 업로드 드롭존 hover/active와 분석 진행 바·팁 카드가 일반 파랑(`blue-500/600`)으로 칠해져 브랜드 톤에서 벗어나 있었다.

- `UploadZone`: 드롭존 테두리/배경/문구의 `gray→slate`, hover·drag 강조색 `blue→brand`
- `AnalysisProgress`: 진행 바·단계 텍스트·팁 카드 `gray→slate`, `blue→brand`
- `ClauseCard`: 본문/설명/권고/법령근거 박스 `gray→slate`, `blue→brand`
- `SkeletonLoader`, `CircularCountdown`: 스켈레톤·카운트다운 `gray→slate`, 기본 색 `#2563EB→#0F2B5B`
- `MyPage`: 로딩 스피너 `blue→brand`, 계약유형 배지 `indigo-600/gray-400 → accent-500/slate-400`
- `PaymentPage`: 토스페이 칩 배경/테두리 색 정합
- `ChecklistPage`: 집계기간 선택 버튼의 `bg-brand-600`인데 `border-blue-600`로 어긋난 테두리 정합

**(B) "안전" 색상 emerald/green 혼용**
디자인 토큰(`risk.safe = green-600`)·대부분 화면은 `green`을 쓰는데 시세 화면(`MarketPage`)·`index.css`의 안전 배지는 `emerald`를 써서 화면 간 같은 "안전"이 다른 초록으로 보였다. → 토큰에 맞춰 **green으로 통일**.

**(C) 플레이스홀더 콘텐츠 노출**
`MyPage` 공지사항 모달이 더미 데이터(`제목/내용 = "임시"`)로 노출되고 있었다. → 정식 오픈/시세 기능 추가 안내 2건의 실제 공지로 교체.

**(D) React Hooks 규칙 위반 (잠재적 크래시)**
`SpecialClausesPage`에서 무료 유저 조기 `return` **이후**에 `useQuery`가 호출되고 있었다. 결제 등으로 `quota`가 바뀌어 유·무료 분기가 전환되면 렌더 간 훅 개수가 달라져 React가 "Rendered more hooks than during the previous render"로 크래시할 수 있는 구조였다. → `useQuery`를 조기 return 앞으로 이동하고 `enabled: !!reportId && !!isPaidUser`로 무료 유저는 요청만 차단.

**(E) 동작하지 않는 버튼 / 어투**
- `LoginPage` 약관 항목의 "보기" 버튼이 핸들러 없는 죽은 버튼 → 피드백 토스트 연결.
- `MyPage` "분석 이력을 긁어오는 중..." 비격식 표현 → "불러오는 중..."으로 정리.

### 2.2 권고 사항 (이번엔 변경하지 않음 — 검토 후 진행 권장)

1. **시세 계산기 코드 중복**: `ChecklistPage`가 `MarketPage`의 실거래가 조회/전세가율 계산 로직(~200줄)을 거의 그대로 중복 보유. 공용 훅(`useMarketQuery`) 또는 컴포넌트로 추출해 분기 방지 권장.
2. **아이콘 언어 혼용**: SVG 라인 아이콘과 이모지 배지가 섞여 있다. 브랜드 완성도를 위해 한쪽으로 통일 검토.
3. **`BottomNavBar` 내부 식별자 혼동**: 가운데 탭 `id:'checklist'`인데 라벨 '시세', 경로 `/market`. 식별자명을 실제 의미에 맞게 정리.
4. **회귀 방지**: ESLint에 "원시 `gray-*`/비브랜드 `blue-*` 금지" 커스텀 규칙을 추가하면 색상 미통일 재발을 막을 수 있다.
5. **운영 전 확인**: 포트원/OAuth 실키는 `VITE_DEMO_MODE` 분기로 가려져 있으므로 프로덕션 환경변수 주입을 배포 체크리스트에서 재확인.

## 3. 검증 상태

- **변경 파일 무결성**: 실제 작업 폴더의 파일을 직접 확인하여 모든 수정이 온전히 반영됨을 검증.
- **타입 체크(tsc)**: 변경으로 인한 타입/구문/식별자 오류 **0건**. (이 세션의 샌드박스 셸이 워크스페이스를 별도 미러로 보며 동기화가 어긋나, 셸에서의 전체 `npm run build`/`vitest`/`eslint` 실행 결과는 신뢰할 수 없었음. 발생한 tsc 오류는 전부 미러 사본의 파일 끝 잘림 아티팩트로, 실제 파일과 무관함.)
- **미실행(환경 제약)**: 라이브 브라우저 시각 점검은 (1) Chrome 확장 미연결, (2) dev 서버가 사용자 PC에서 떠 있어야 접근 가능 — 두 조건이 필요해 이번 세션에서 실행하지 못함.

### 사용자 환경에서 권장 검증 명령

```bash
cd frontend
npm run build      # tsc + vite build
npm run lint
npm run test
# 라이브 점검: 목업으로 단독 구동
VITE_ENABLE_MOCK=true npm run dev   # http://localhost:3000
```

dev 서버를 띄우고 Chrome 확장을 연결해 주시면, 12개 페이지 전체를 브라우저로 띄워 시각 점검까지 마저 진행하겠습니다.
(참고: 브라우저 자동화·computer-use는 현재 macOS 전용이라 Windows 환경에서는 라이브 시각 점검을 대신 수행하지 못했습니다.)

## 4. 후속 리팩터링 (권고 항목 반영)

평가 시 "권고"로 남겼던 항목을 이어서 적용했다.

**(1) 시세 조회 로직 중복 제거**
`MarketPage`와 `ChecklistPage`에 거의 동일하게 중복돼 있던 시세 조회 흐름(지역/동 선택 → 실거래가 summary 조회 → 전세가율 계산)과 금액 포매터를 단일 출처로 통합했다.

- 신규 `src/hooks/useMarketQuery.ts` — 시세 상태·effect·`handleFetchMarket`를 캡슐화한 공용 훅
- 신규 `src/utils/marketFormat.ts` — `formatKrw`/`formatNumber`/`formatPeriod` 공용 유틸
- 두 페이지는 `const { ... } = useMarketQuery()` 한 줄로 소비하도록 변경(JSX는 그대로). 페이지당 ~140줄의 중복 제거. 미사용 변수(`propertyNum`)·임포트까지 정리.

**(2) 네비게이션 식별자 정리 + 색상 회귀 방지 가드**
- `BottomNavBar`의 혼동되는 내부 id `'checklist'`(라벨 '시세', 경로 `/market`)를 `'market'`으로 정리.
- 신규 `frontend/scripts/check-tailwind-colors.mjs` — 레거시 `gray-*`/`indigo-*`/`emerald-*` 및 허용 목록 외 `blue-*` 사용을 차단하는 검사 스크립트. `npm run lint`에 연결(`lint:colors`로 단독 실행도 가능). 정당한 예외는 해당 줄에 `color-guard-ignore` 주석으로 허용. (allowlist: Toast=info, PaymentPage=토스페이, MarketPage=월세 '저렴' 정보, DevPanel)

**(3) 아이콘 언어 통일 (보수적 적용)**
규칙: **CTA 버튼은 텍스트 전용**(장식 이모지 제거), **상태 배지·빈 상태·브랜드 마크(카카오/토스 등)·링크 아이콘의 이모지는 유지**해 친근한 톤을 보존. 결과/특약 플로우의 CTA(특약사항 추천, 체크리스트 보기, 복사/수정, 전체 PDF 다운로드)에서 선행 이모지를 제거. 전면적인 SVG/이모지 통일은 시각 반복 검토가 필요하므로 이번 범위에서 제외.

> 검증 한계는 §3과 동일하다. 신규 훅/유틸은 색상이 없어 가드와 무관하며, 변경된 두 페이지는 댕글링 참조·미사용 심볼이 없음을 정적 검색으로 확인했다. 최종 확인은 사용자 환경에서 `npm run build && npm run lint && npm run test` 권장.
